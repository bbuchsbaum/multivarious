<task type="auto">
  <name>Task 1: Submit package to CRAN</name>
  <files></files>
  <action>
Execute the CRAN submission using devtools:

1. Open R in the package directory

2. Run submission command:
   ```r
   devtools::submit_cran()
   ```

3. This will:
   - Build the package tarball with pkgbuild::build(manual = TRUE)
   - Upload to CRAN's web form
   - Populate form fields from DESCRIPTION and cran-comments.md
   - Create CRAN-SUBMISSION file recording the submission

4. Watch for any errors during upload. If upload fails, report the error.

5. Upon successful upload, devtools will print a message indicating the package was uploaded.

Note: This step is automated. The human action (clicking email link) comes next.
  </action>
  <verify>
Console output indicates successful upload to CRAN.
CRAN-SUBMISSION file is created in package root.
  </verify>
  <done>Package tarball uploaded to CRAN submission form.</done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Confirm CRAN submission via email</action>
  <why>CRAN requires email confirmation to complete submission. This link is sent to the maintainer email and cannot be clicked programmatically.</why>
  <instructions>
1. Check the inbox for brad.buchsbaum@gmail.com

2. Look for email from CRAN with subject like "CRAN submission multivarious 0.3.0"

3. Click the confirmation link in the email

4. The link will confirm the submission was received by CRAN

5. Note: The package is NOT submitted until this link is clicked. If the email doesn't arrive within 30 minutes, check spam folder or re-run submit_cran().
  </instructions>
  <resume-signal>Type "confirmed" once you have clicked the email confirmation link</resume-signal>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>CRAN submission completed and confirmed</what-built>
  <how-to-verify>
1. Visit https://cran.r-project.org/incoming/ in your browser

2. Look for "multivarious" in one of the queue folders:
   - incoming/
   - inspect/
   - pretest/
   - pending/

3. The package name should appear with version 0.3.0

4. Alternatively, run in R:
   ```r
   foghorn::cran_incoming("multivarious")
   ```
   (Install foghorn first if not available: install.packages("foghorn"))

Note: It may take a few minutes to several hours for the package to appear in the queue after confirmation.
  </how-to-verify>
  <resume-signal>Type "in-queue" once you see multivarious in the CRAN incoming queue, or "not-visible" if not appearing after 1+ hours</resume-signal>
</task>
