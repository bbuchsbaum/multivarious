  <files>R/pca.R, R/cPCA.R, R/all_generic.R</files>
  <action>
Review and convert \\dontrun{} blocks that contain executable code:

1. **R/pca.R lines 639-646** (biplot.pca example):
   - This example uses ggrepel for text repelling
   - It IS executable if ggrepel is installed
   - Change `\\dontrun{` to `\\donttest{`

2. **R/cPCA.R lines 118-129** (cPCAplus plot example):
   - This is a graphics example that plots results
   - It IS executable
   - Change `\\dontrun{` to `\\donttest{`

3. **R/all_generic.R lines 696-734** (perm_test examples):
   - These examples use MASS::lda and cross_projector
   - They ARE executable but may take time
   - Change `\\dontrun{` to `\\donttest{`

The rule: Use `\\dontrun{}` ONLY for code that would:
- Produce errors intentionally
- Write to user's file system
- Require external credentials/accounts

Use `\\donttest{}` for code that:
- Takes a long time
- Requires optional packages (CRAN skips these during regular checks)
- Uses significant memory
  </action>

Run: `grep -n "dontrun" R/pca.R R/cPCA.R R/all_generic.R`
Expected: No matches (all converted to donttest)
Run: `grep -n "donttest" R/pca.R R/cPCA.R R/all_generic.R`
Expected: Three matches showing the converted examples
  </verify>
  <done>All slow-but-executable examples use \\donttest{} instead of \\dontrun{}</done>
