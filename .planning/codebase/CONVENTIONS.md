# Coding Conventions

**Analysis Date:** 2026-01-20

## Naming Patterns

**Files:**
- Snake_case for source files: `pre_process.R`, `bi_projector.R`, `all_generic.R`
- Hyphenated for related utilities: `preproc-generics.R`, `preproc-utils.R`
- Test files: `test_<component>.R` (e.g., `test_pca.R`, `test_projector.R`)

**Functions:**
- Snake_case for public functions: `svd_wrapper()`, `partial_project()`, `inverse_projection()`
- Dot notation for S3 methods: `project.projector()`, `print.pca()`, `truncate.bi_projector()`
- Internal helpers prefixed with dot: `.normalize_which()`, `.invert_which_recip()`, `.select_eigs()`

**Variables:**
- Short descriptive names: `v` (coefficients), `s` (scores), `X` (data matrix), `Y` (response)
- Underscores for multi-word: `new_data`, `block_indices`, `col_means`
- Loop indices: `i`, `j`, `k`, `a` (for component indices)

**Classes:**
- Snake_case for S3 classes: `projector`, `bi_projector`, `cross_projector`, `pre_processor`
- Compound names describe inheritance: `partial_projector`, `composed_projector`, `multiblock_projector`

## Code Style

**Formatting:**
- No explicit formatter tool detected
- Indentation: 2 spaces
- Line width: Generally 80-100 characters
- Opening braces on same line as function/control statement

**Linting:**
- No `.eslintrc` equivalent detected
- Relies on R CMD check for style validation
- roxygen2 for documentation enforcement

**Spacing:**
- Space after commas in argument lists
- Space around assignment operators (`<-`, `=`)
- No space before parentheses in function calls
- Space after `#` in comments

## Import Organization

**Order:**
1. Base R functions (stats, methods, utils)
2. Matrix operations (Matrix package)
3. Statistical packages (MASS, glmnet, pls)
4. Utility packages (chk, rlang, assertthat)
5. Visualization (ggplot2, ggrepel, crayon)

**Package Imports:**
- Use `@importFrom` for specific functions: `@importFrom Matrix crossprod`
- Use `@import` for heavily-used packages: `@import chk`, `@import ggplot2`
- Access via `::` for optional dependencies: `corpcor::pseudoinverse()`, `RSpectra::svds()`

**Namespace Management:**
- All exports defined in NAMESPACE (auto-generated by roxygen2)
- Generic functions exported from `all_generic.R`
- S3 methods registered via `@export` tag

## Error Handling

**Validation Patterns:**
```r
# Use chk package for input validation
chk::chk_matrix(X)
chk::chk_numeric(comp)
chk::chk_range(ncomp, c(1, ncomp(x)))
chk::chk_equal(ncol(new_data), length(colind))
chk::chk_s3_class(preproc, "pre_processor")
chk::chk_not_null(x$colind)

# Use chkor_vld for either/or checks
chk::chkor_vld(chk::vld_matrix(X), chk::vld_s4_class(X, "Matrix"))
```

**Error Messages:**
```r
# Use stop() with informative messages
stop("Requested ncomp must be between 1 and ", old_ncomp)
stop(sprintf("Cannot compute %d singular vectors for %d x %d matrix.", k, nrow(M), ncol(M)))

# Use cli_abort for structured errors (newer pattern)
cli::cli_abort(c(
  "!" = "{.fn transfer} is not implemented for {.cls {class(x)[1]}}.",
  "i" = "Provide a method or ensure the object inherits from a supported class."
))

# Use warning() for non-fatal issues
warning("Some variance values are near zero; results may be unstable.")
```

**Try-Catch Pattern:**
```r
res <- tryCatch({
  svd(Xp, nu = k, nv = k, ...)
}, error = function(e) {
  stop(sprintf("SVD computation failed for method '%s': %s", method, e$message), call. = FALSE)
})
```

## Function Design

**Size:** Functions typically 20-100 lines; larger functions for complex algorithms (e.g., `perm_test.pca` ~250 lines)

**Parameters:**
- Required parameters first, optional with defaults after
- Use `...` for pass-through to underlying methods
- Use `match.arg()` for string options:
```r
method <- match.arg(method)
alternative <- match.arg(alternative)
```

**Return Values:**
- Return S3 objects with consistent structure
- Use `invisible(x)` for print methods
- Return lists for multiple outputs: `list(preproc = fitted_preproc, transformed = transformed)`

**Default Values:**
```r
# Sensible defaults derived from data
ncomp = min(dim(X))
preproc = center()
method = c("fast", "base", "irlba", "propack", "rsvd", "svds")

# Small regularization defaults
lambda = 1e-6
tol = .Machine$double.eps
```

## Module Design

**Exports:**
- All public functions have `@export` tag
- S3 methods exported via `@export` on method definition
- Generic functions defined in `all_generic.R`, methods in respective class files

**Barrel Files:**
- `all_generic.R` serves as central generic function registry
- `preproc-generics.R` for preprocessing API generics
- No formal barrel/index file pattern for exports

**Class Constructors:**
```r
# Constructor pattern - returns object with class attribute
projector <- function(v, preproc = prep(pass()), ..., classes = NULL) {
  # Input validation
  if (!is.matrix(v)) v <- as.matrix(v)
  chk::chk_s3_class(preproc, "pre_processor")

  # Build object structure
  out <- structure(
    list(v = v, preproc = preproc, ...),
    class = c(classes, "projector")
  )

  # Add cache environment
  attr(out, ".cache") <- new.env(parent = emptyenv())
  out
}
```

**Caching Strategy:**
```r
# Cache expensive computations in .cache environment
cache_env <- attr(x, ".cache")
use_caching <- !is.null(cache_env) && is.environment(cache_env)
key <- "inv_proj"

if (use_caching && !is.null(cache_env[[key]])) {
  return(cache_env[[key]])
}

# Compute and cache
result <- expensive_computation()
if (use_caching) {
  cache_env[[key]] <- result
}
```
